# Mini NPU Simulator

숫자 행렬로 표현된 패턴이 어떤 특징(Cross, X, O)에 가까운지 수학적으로 판별하는 Python 기반 NPU 시뮬레이터 프로젝트임.

이 프로젝트의 핵심은 가중치 모델과 데이터의 곱셈-누적 합이라는 개념에 기반함. 기하학적으로 **위치가 겹칠수록 곱셈 결과가 1로 보존되어 누적 점수가 상승하는 유사성 원리**를 구현한 것임.
1. 입력된 형태를 2차원 숫자 행렬로 맵핑함
2. 입력 패턴과 기준 필터들의 같은 위치 요소 값을 서로 곱함 (Multiply)
3. 곱해진 값들을 모두 더해 최종 유사도 점수를 만듦 (Accumulate)
4. 필터별 연산된 점수를 비교하여 가장 높은 점수를 가지는 판정 라벨을 예측함

컴퓨터의 구조 중 이 연산(MAC 연산) 구조에 가장 강점을 가진 것이 **NPU(Neural Processing Unit)**임. 본 프로젝트는 NPU의 물리적 하드웨어 파이프라인 처리를 일반 CPU 상에서 모방(Simulate)하며 연산 크기 증가에 따른 성능 변화를 체험하는 것을 주요 목적으로 설계됨.

## 🛠 실행 방법
터미널 환경 격리 및 실행 일관성을 위해 Docker Compose를 활용하여 구동함.

```bash
docker compose run --rm npu-app
```

### [모드 1] 사용자 입력 상호작용 모드
- 사용자가 직접 3x3 필터(A, B, C)와 패턴 배열을 입력해서 MAC 계산 결과를 체험하는 모드.
- **예외 방어막**: 데이터 입력 시 행/열 개수 불일치 또는 숫자 파싱 실패가 감지되면, 내부 `ValueError` 예외 처리를 통해 즉시 오류를 안내하고 재입력을 유도함.
- **판독 신뢰성**: 허용 오차(Epsilon: 1e-9) 내 동점이 발생하면 치우침 없이 `UNDECIDED`(판정 불가)를 선언하여 AI 모델 판독의 False Positive를 완벽하게 방어함.

### [모드 2] 자동 데이터 JSON 분석 채점 모드
- `data.json`의 필터와 패턴(`size_N_idx`)을 로드하여 일괄 자동 채점을 진행함 (동점 시 `UNDECIDED` 처리 후 엄격한 잣대로 시스템상 명시적 FAIL로 집계됨).
- 스크립트가 패턴 키를 `split('_')`으로 파싱하여 인덱스(N)를 추출한 뒤, 동적 크기에 알맞은 필터 묶음을 매핑해 찾아가는 **사이즈 매칭 스마트 라우팅 알고리즘**을 포함함.
- 연산 종료 후 총 테스트 횟수, 통과/실패 지표 및 상세 실패 사유 요약 지표를 화면 로거에 출력함.

## 💡 개발 제약 조건과 최적화 (O(N²) 한계 돌파)
"NumPy 등 외부 라이브러리 엄격 금지" 제약을 돌파하기 위해 파이썬 내장 기능을 극한까지 끌어올림.

### 1D 캐시 친화적 파이프라인 최적화 (Memory Flattening)
CPU 단일 스레드 순차 연산의 치명적인 병목 현상을 타파하고자 다차원 데이터 탐색 루프를 걷어내고, 메모리 주소를 **1차원으로 평탄화(Flatten)**함. 이후 파이썬 내장 C-level 언패킹 수단인 `zip() -> sum()` 연결 연산을 배치함. 그 결과 CPU 환경 상에서도 대규모 패턴 처리 시 발생하는 지연(NPU 병렬 캐시 부재) 한계를 비약적으로 극복함.

### 유연한 필터 확장성 및 데이터 정규화 모듈
- 기존 Cross, X 필터 외에 **'O' 필터(테두리 모델)를 새롭게 확장 적용**하였음. 데이터 구조 확장 ➡️ 파서 룰 추가 ➡️ 로직 전개라는 3단계 프로세스를 준수하여 새로운 라벨이 무한히 유입되더라도 수용 가능하도록 아키텍처 결합도를 낮춤.
- 라벨 정규화(`normalize_label`)를 별도 함수/모듈로 완전히 격리 분리함. 예상치 못한 `+`, `cross`, `x` 등 오기재된 입력은 중앙 엔진에서 일괄 표준 치환하여 확장성과 안정성을 동시에 보장함.

## 📊 결과 리포트 및 실패(FAIL) 케이스 진단

데이터 분석 시 로그에 나타나는 5종의 `FAIL` 리포트는 시스템 버그가 아니라, **의도적 함정 시나리오 및 구조 파손을 시스템 스코어링 모터가 완벽하게 간파해 낸 자랑스러운 기록**임.

1. **오기재 감지 로직 정상 가동 (size_5_5)**
   - 입력된 배열 형상은 명백한 `X` 구조라 X 판독 점수(9점)가 도출되었으나, 정답지(`expected`)가 의도적인 오기재 함정(`+`)으로 구성되어 있어 FAIL이 도출됨. 시뮬레이터 스스로의 식별 능력이 완벽함을 객관적으로 증명하는 케이스.
2. **크기 불일치 쉐도우 방어망 작동 성공 (size_5_6)**
   - size_5 규격 선언 구역에 4행짜리 파손 데이터가 은닉되어 있었으나, `validate_matrix_size` 이중 검증 로직이 안전하게 튕겨내어 런타임 종료를 막고 예외처리(`size_mismatch`)로 연착륙시킴.
3. **이진 부동소수점 오차 방어 및 볼륨 한계 노출 (size_5_3 등)**
   - 소수 데이터 기반의 도면 노이즈 입력 시, 물리적인 절삭 오차가 발생하므로 비교 정책(`abs(A - B) < 1e-9`)의 설계 타당성이 즉시 증명됨. 
   - 노이즈가 모두 깔린 화면에서는 선분 점유 면적이 넓은 새로운 O 필터가 수학적 내적에 의해 가장 큰 점수를 가져가는 현상이 발생함. 이는 향후 거대 사이즈 진행 시 Weight Normalization(활성화 칸 수로 점수 정규화)의 설계 필요성을 암시함.

## ⏱ 성능 측정 및 시간 복잡도(O(N²)) 분석 결과
성능 분석 표 대시보드에는 **입력 크기 단위(N)**, **평균 소요 시간(ms)**, **연산 횟수(ops_count: N²)** 지표가 정밀하게 집계되어 시각화 됨.
- 측정 시 파일 I/O나 화면 표출 로직에 의한 간섭 지연을 철저하게 배제하기 위해, 수학적 핵심 연산 구간(`calculate_mac`) 직전과 직후에만 `time.perf_counter()`를 밀착시켜 순수한 병목 "연산 구간"만을 추출함.
- 행렬 변의 크기(N)가 5배 커질 때(예: 5x5에서 25x25), 단일 위치 단위 스캐닝 방문 면적이 비례하는 제곱 면적으로 늘어남. 결과적으로 통계 도표 상 연산 수행 횟수와 전체 처리 시간의 폭이 약 25배 단위로 폭증하는 전형적인 **O(N²) 스케일링(시간 복잡도)** 경향성 법칙을 정확하게 입증함.
